# DeltaAegis v0.45 NetSniper Context Consumer

## Purpose

DeltaAegis consumes the additive `deltaaegis_context` object emitted inside
NetSniper's existing classification projection. The complete classification
object was already persisted in `asset_observations.classification_json`, so
this checkpoint requires no database migration and no ingestion change.

## Read boundary

The context is decoded only for the latest observation returned by the existing
`/api/asset` detail payload. The shared `/api/assets` list serializer remains
unchanged so large inventory responses do not carry full evidence-axis detail.

The additive detail fields are:

- `classification_context_available`: true only for a non-empty object;
- `classification_context`: the exact NetSniper object, or an empty object when
  absent, malformed, empty, or incorrectly typed.

## Compatibility and ownership

DeltaAegis does not recalculate NetSniper classifications and does not interpret
`operator_disposition`, axis confidence, uncertainty, or semantic fingerprints.
This checkpoint does not change risk scoring, events, alerts, reports, lifecycle
state, the legacy NetSniper intelligence table, or dashboard rendering.

Older bundles remain compatible because missing or malformed context fails
closed to `classification_context_available=false` and
`classification_context={}`.

## Validation

Run:

```bash
python3 tools/validate_v0_45_netsniper_context_consumer.py
```

The audited source state did not yet contain a v0.45 full release gate. This
validator should be added to that flat gate when the v0.45 release gate is
established; this checkpoint does not create or modify one implicitly.
