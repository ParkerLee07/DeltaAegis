# DeltaAegis

**Self-hosted, delta-first network-state monitoring console.**

DeltaAegis ingests immutable telemetry bundles generated by NetSniper, stores normalized historical snapshots in SQLite, and explains how a monitored network changes over time.

The project is intentionally narrow: it focuses on reliable attack-surface change detection before expanding into broader SIEM capabilities.

## Current Release

```text
DeltaAegis v0.2
```

## Features

- Ingests finalized `netsniper-run-v1` and `netsniper-run-v2` bundles.
- Parses Nmap XML for neutral service telemetry.
- Uses archived neighbor-table enrichment for MAC-backed local identity correlation.
- Separates globally administered MACs, locally administered MACs, and IP-only observations.
- Filters unusable addresses such as subnet network and broadcast addresses.
- Preserves historical snapshots and append-only JSONL events.
- Applies snapshot-quality and scan-profile compatibility gates.
- Tracks stable-asset lifecycle state with a three-accepted-scan removal threshold.
- Emits service-opened, service-closed, NetSniper finding, profile-reset, and identity lifecycle events.
- Maintains operator-facing alerts with `OPEN`, `ACKNOWLEDGED`, `RESOLVED`, and `SUPPRESSED` states.
- Provides an interactive terminal menu and automation-friendly direct commands.

## Architecture

```text
NetSniper full-pipeline run
        ↓
Immutable telemetry bundle
        ↓
DeltaAegis ingestion and normalization
        ↓
SQLite historical snapshot store
        ↓
Profile compatibility and quality gates
        ↓
Delta engine
        ↓
Append-only events + stateful operator alerts
```

See [`Docs/architecture.md`](Docs/architecture.md).

## Requirements

- Linux
- Python 3.10 or newer
- NetSniper with immutable telemetry bundles enabled

DeltaAegis uses Python standard-library components for its current prototype and does not require a separate database server.

## Installation

```bash
git clone https://github.com/ParkerLee07/DeltaAegis.git
cd DeltaAegis
chmod +x install.sh
./install.sh
```

## Usage

Launch the interactive console:

```bash
deltaaegis
```

Direct commands remain available for automation:

```bash
deltaaegis ingest
deltaaegis summary
deltaaegis snapshots --limit 20
deltaaegis events --limit 50
deltaaegis alerts --status OPEN --limit 50
deltaaegis health --limit 20
deltaaegis asset 192.168.4.32
deltaaegis latest
deltaaegis paths
```

If a NetSniper scan profile changes, review the quarantined snapshot and approve a new baseline explicitly:

```bash
deltaaegis approve <scan_id>
```

## Runtime Data

Runtime state remains local and is excluded from Git:

```text
~/DeltaAegis/
├── data/deltaaegis.db
├── events/events.jsonl
└── backups/
```

NetSniper telemetry remains under:

```text
~/NetSniper/runs/
```

## Scope and Limitations

DeltaAegis `v0.2` is a network-state monitoring prototype, not a replacement for an enterprise SIEM. It does not ingest endpoint logs, retain full packets indefinitely, or use machine-learning anomaly detection. Its conclusions are limited to the telemetry provided by its sensors.

## Authorized Use Only

Use DeltaAegis only on networks and systems for which you have explicit authorization.
