# DeltaAegis

**Self-hosted, delta-first network-state monitoring and investigation console.**

DeltaAegis ingests immutable telemetry bundles generated by NetSniper, stores normalized historical snapshots in SQLite, and explains how a monitored network changes over time.

The project is intentionally narrow: it focuses on reliable attack-surface change detection, asset-state tracking, alert review, and Markdown investigation reporting before expanding into broader SIEM capabilities.

## What DeltaAegis Does

DeltaAegis compares accepted NetSniper snapshots over time and turns network changes into structured events, alerts, and reports.

```text
NetSniper telemetry bundle
        ↓
DeltaAegis ingestion
        ↓
SQLite snapshot history
        ↓
Delta engine
        ↓
Events + alerts
        ↓
Asset context + analyst notes
        ↓
Markdown investigation reports
```

DeltaAegis is designed to help answer questions like:

* What changed on the network?
* Which services appeared or disappeared?
* Which assets are stable, missing, or newly observed?
* Which alerts have been reviewed?
* Why did an analyst acknowledge or suppress an alert?
* What asset owner, role, or criticality is associated with a finding?

---

## Features

* Ingests finalized `netsniper-run-v1` and `netsniper-run-v2` bundles.
* Parses Nmap XML for neutral service telemetry.
* Uses archived neighbor-table enrichment for MAC-backed local identity correlation.
* Separates globally administered MACs, locally administered MACs, and IP-only observations.
* Filters unusable addresses such as subnet network and broadcast addresses.
* Preserves historical snapshots and append-only JSONL events.
* Applies snapshot-quality and scan-profile compatibility gates.
* Tracks stable-asset lifecycle state with a three-accepted-scan removal threshold.
* Emits service-opened, service-closed, NetSniper finding, profile-reset, and identity lifecycle events.
* Maintains operator-facing alerts with `OPEN`, `ACKNOWLEDGED`, `RESOLVED`, and `SUPPRESSED` states.
* Supports alert review notes with analyst-provided reasons.
* Supports asset owner, role, criticality, and notes.
* Generates Markdown investigation reports.
* Includes asset context and alert review notes in reports.
* Provides an interactive terminal menu and automation-friendly direct commands.

## Architecture

```text
NetSniper full-pipeline run
        ↓
Immutable telemetry bundle
        ↓
DeltaAegis ingestion and normalization
        ↓
SQLite historical snapshot store
        ↓
Profile compatibility and quality gates
        ↓
Delta engine
        ↓
Append-only events + stateful operator alerts
        ↓
Asset annotations + alert review notes
        ↓
Markdown investigation reports
```

See [`Docs/architecture.md`](Docs/architecture.md) for deeper architecture notes.

## Requirements

* Linux
* Python 3.10 or newer
* NetSniper with immutable telemetry bundles enabled

DeltaAegis currently uses Python standard-library components and SQLite. It does not require a separate database server.

Optional but useful:

* `sqlite3` CLI for manual database inspection
* Git for source control and updates

## Installation

```bash
git clone https://github.com/ParkerLee07/DeltaAegis.git
cd DeltaAegis
chmod +x install.sh
./install.sh
```

After installation, launch DeltaAegis with:

```bash
deltaaegis
```

You can also run it directly from the repository:

```bash
python3 deltaaegis.py
```

## Usage

Launch the interactive console:

```bash
deltaaegis
```

or:

```bash
python3 deltaaegis.py menu
```

Direct commands are available for automation and repeatable workflows.

---

## Core Commands

Ingest new NetSniper telemetry bundles:

```bash
deltaaegis ingest
```

Show system summary:

```bash
deltaaegis summary
```

List imported snapshots:

```bash
deltaaegis snapshots --limit 20
```

Show recent delta events:

```bash
deltaaegis events --limit 50
```

Filter events by severity:

```bash
deltaaegis events --severity HIGH --limit 50
```

Filter events by type:

```bash
deltaaegis events --event-type MONITORED_SERVICE_OPENED --limit 50
```

Show alerts:

```bash
deltaaegis alerts --status OPEN --limit 50
```

Show snapshot health:

```bash
deltaaegis health --limit 20
```

Show latest accepted snapshot:

```bash
deltaaegis latest
```

Show configured telemetry paths:

```bash
deltaaegis paths
```

## Baseline Review

If a NetSniper scan profile changes, DeltaAegis can quarantine the snapshot for review instead of automatically treating it as a normal baseline continuation.

Approve a reviewed snapshot as a new baseline:

```bash
deltaaegis approve <scan-id>
```

Example:

```bash
deltaaegis approve 20260612-162354
```

## Asset Investigation

Show asset history by asset key or current IP:

```bash
deltaaegis asset 192.168.4.32
```

Show timeline for a specific subject key:

```bash
deltaaegis asset-timeline '<subject-key>'
```

Examples:

```bash
deltaaegis asset-timeline 'ip:192.168.4.32'
deltaaegis asset-timeline '192.168.4.32:tcp/8080'
deltaaegis asset-timeline 'mac:aa:bb:cc:dd:ee:ff'
```

Filter an asset timeline by severity:

```bash
deltaaegis asset-timeline '<subject-key>' --severity HIGH
```

---

## Alert Review

Acknowledge an alert:

```bash
deltaaegis ack <alert-id> --reason "Reviewed and confirmed expected behavior"
```

Suppress an alert:

```bash
deltaaegis suppress <alert-id> --reason "Known recurring lab service"
```

Show detailed alert context:

```bash
deltaaegis alert-detail <alert-id>
```

Show review notes for an alert:

```bash
deltaaegis alert-notes <alert-id>
```

Alert review notes preserve analyst decisions over time. This makes it easier to understand why an alert was acknowledged or suppressed.

## Asset Annotations

DeltaAegis supports lightweight asset annotations for owner, role, criticality, and notes.

Annotate an asset:

```bash
deltaaegis annotate-asset '<asset-key>' \
  --owner "IT" \
  --role "Printer" \
  --criticality "LOW" \
  --notes "Known office printer"
```

Show notes for one asset:

```bash
deltaaegis asset-notes '<asset-key>'
```

Show annotation history for one asset:

```bash
deltaaegis asset-notes '<asset-key>' --history
```

List saved asset annotations:

```bash
deltaaegis asset-annotations
```

Limit annotation output:

```bash
deltaaegis asset-annotations --limit 10
```

Asset annotations are used by investigation reports when an event or alert subject matches a saved annotation.

## Investigation Reports

Generate a Markdown investigation report:

```bash
deltaaegis report
```

Write a report to a specific output path:

```bash
deltaaegis report --output reports/example-report.md
```

Generate a report using the latest relevant events:

```bash
deltaaegis report --latest
```

Filter a report by severity:

```bash
deltaaegis report --severity HIGH
```

Limit report event count:

```bash
deltaaegis report --limit 100
```

Reports can include:

* Report metadata
* Event totals
* Severity counts
* Event type counts
* Open alert context
* Annotated asset context
* Per-event asset context
* Alert review notes
* Recommended follow-up guidance

This allows DeltaAegis to produce a readable investigation artifact instead of only terminal output.


```text
~/NetSniper/runs/
```

## Data Model

DeltaAegis stores its local state in SQLite.

Core data includes:

```text
snapshots
asset_observations
service_observations
finding_observations
delta_events
asset_lifecycle
alerts
alert_notes
asset_annotations
asset_annotation_history
```

The `delta_events` table preserves the append-only event history.

The `alerts` table maintains stateful operator-facing alerts.

The `alert_notes` table records analyst review decisions.

The `asset_annotations` and `asset_annotation_history` tables preserve asset owner and context information.

## Scope and Limitations

DeltaAegis `v0.3.5` is a network-state monitoring and investigation prototype.

It is not a replacement for an enterprise SIEM.

DeltaAegis does not currently:

* Ingest endpoint logs
* Store full packet captures indefinitely
* Perform machine-learning anomaly detection
* Send email or chat notifications
* Provide a web dashboard
* Automatically discover business owners for assets
* Replace manual analyst review

Its conclusions are limited to the telemetry provided by NetSniper and the local DeltaAegis database.

Use DeltaAegis only on networks and systems for which you have explicit authorization.
