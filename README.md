## DeltaAegis v0.8.5 Current Capabilities

DeltaAegis v0.8.5 is a snapshot-based network-state monitoring and investigation console that uses NetSniper telemetry to track assets, compare accepted scans, surface meaningful changes, and prioritize follow-up work.

### What v0.8.5 adds

- **NetSniper classification intelligence ingestion** — stores evidence-backed classification data from NetSniper bundles, including type, primary type, decision, confidence, evidence, contradictions, and candidate roles.
- **Dashboard intelligence visibility** — adds classification columns to asset inventory, a NetSniper Intelligence panel on asset detail pages, and dashboard-level NetSniper Intelligence Summary cards.
- **Classification review workflow** — highlights classified, possible/weak, unknown, evidence-backed, contradictory, and high-confidence assets.
- **Report intelligence summaries** — generated Markdown investigation reports now include a NetSniper Intelligence Summary, top classifications, classification review queue, and role-aware recommended actions.
- **Classification-aware risk context** — risk scoring now includes conservative, explainable role-aware context for suspected printers, cameras/NVRs, web servers, domain controllers, databases, container infrastructure, and unknown assets.
- **Role-aware recommended actions** — DeltaAegis recommends different follow-up actions depending on suspected asset role, exposed services, confidence level, and whether the role is confirmed, possible, or unknown.
- **Recommendation wording polish** — suspected roles are no longer described as generic unknown assets.

### Current investigation flow

1. Run NetSniper and generate a telemetry bundle.
2. Ingest the bundle into DeltaAegis.
3. DeltaAegis stores the accepted snapshot, updates asset lifecycle state, and compares the scan to prior accepted baselines.
4. Dashboard views show asset state, service deltas, alerts, risk subjects, classification intelligence, and recommended next actions.
5. Reports summarize network scope, asset lifecycle, classification intelligence, top risks, role-aware recommendations, events, alerts, and next steps.

DeltaAegis remains intentionally conservative. It exposes confidence, evidence, contradictions, and review targets so operators can validate uncertain assets.

**Self-hosted, delta-first network-state monitoring and investigation console.**

DeltaAegis ingests immutable telemetry bundles generated by NetSniper, stores normalized historical snapshots in SQLite, and explains how a monitored network changes over time.

The project focuses on reliable attack-surface change detection, asset-state tracking, alert review, risk prioritization, Markdown investigation reporting, and a read-only local dashboard before expanding into broader SIEM capabilities.

## What DeltaAegis Does

DeltaAegis compares accepted NetSniper snapshots over time and turns network changes into structured events, alerts, and reports.

~~~text
NetSniper telemetry bundle
        ↓
DeltaAegis ingestion
        ↓
SQLite snapshot history
        ↓
Delta engine
        ↓
Events + alerts
        ↓
Asset context + analyst notes
        ↓
Risk prioritization
        ↓
Markdown investigation reports + read-only dashboard
~~~

DeltaAegis helps answer:

- What changed on the network?
- Which services appeared or disappeared?
- Which assets are stable, missing, or newly observed?
- Which alerts have been reviewed?
- Why did an analyst acknowledge or suppress an alert?
- What asset owner, role, or criticality is associated with a finding?
- Which subjects should be reviewed first based on explainable risk scoring?

## Features

- Ingests finalized `netsniper-run-v1` and `netsniper-run-v2` bundles.
- Parses Nmap XML for service telemetry.
- Uses archived neighbor-table enrichment for MAC-backed local identity correlation.
- Separates globally administered MACs, locally administered MACs, and IP-only observations.
- Filters unusable addresses such as network and broadcast addresses.
- Preserves historical snapshots and append-only JSONL events.
- Applies snapshot-quality and scan-profile compatibility gates.
- Tracks stable-asset lifecycle state with a three-accepted-scan removal threshold.
- Emits service-opened, service-closed, NetSniper finding, profile-reset, and identity lifecycle events.
- Maintains operator-facing alerts with `OPEN`, `ACKNOWLEDGED`, `RESOLVED`, and `SUPPRESSED` states.
- Supports alert review notes with analyst-provided reasons.
- Supports asset owner, role, criticality, and notes.
- Generates Markdown investigation reports.
- Includes asset context and alert review notes in reports.
- Calculates an explainable risk register for prioritized analyst review.
- Provides a read-only local dashboard for scans, deltas, alerts, risk, annotations, and identity context.
- Provides an interactive terminal menu and automation-friendly direct commands.

### Network Scope Isolation

DeltaAegis v0.5.1 introduced canonical network scope isolation across scan history, baseline selection, lifecycle state, CLI views, and dashboard views.

Examples:

    deltaaegis scopes
    deltaaegis snapshots --scope 192.168.4.0/24
    deltaaegis latest --scope 192.168.4.0/24
    deltaaegis events --scope 192.168.4.0/24
    deltaaegis alerts --scope 192.168.4.0/24
    deltaaegis risk --scope 192.168.4.0/24
    deltaaegis dashboard --scope 192.168.4.0/24

Network scopes are normalized from valid CIDR targets, so targets such as `192.168.4.25/24` are treated as `192.168.4.0/24`. This prevents snapshots from unrelated subnets, such as `192.168.4.0/24` and `192.168.5.0/24`, from being compared as if they were the same environment.

The dashboard also exposes subnet-aware API endpoints:

    /api/scopes
    /api/summary?scope=192.168.4.0/24
    /api/scan-context?scope=192.168.4.0/24
    /api/events?scope=192.168.4.0/24
    /api/alerts?scope=192.168.4.0/24
    /api/risk?scope=192.168.4.0/24
    /api/annotations?scope=192.168.4.0/24

## Baseline Review

If a NetSniper scan profile changes, DeltaAegis can quarantine the snapshot for review instead of automatically treating it as a normal baseline continuation.

Approve a reviewed snapshot as a new baseline:

~~~bash
deltaaegis approve SCAN_ID
~~~

Example:

~~~bash
deltaaegis approve 20260612-162354
~~~

## Alert Review

Acknowledge an alert:

~~~bash
deltaaegis ack ALERT_ID --reason "Reviewed and confirmed expected behavior"
~~~

Suppress an alert:

~~~

~~~bash
deltaaegis alert-detail ALERT_ID
~~~

Show review notes for an alert:

~~~bash
deltaaegis alert-notes ALERT_ID
~~~

Alert review notes preserve analyst decisions over time. This makes it easier to understand why an alert was acknowledged or suppressed.

## Asset Annotations

DeltaAegis supports lightweight asset annotations for owner, role, criticality, and notes.

Annotate an asset:

~~~bash
deltaaegis annotate-asset 'ASSET_KEY' \
  --owner "IT" \
  --role "Printer" \
  --criticality "LOW" \
  --notes "Known office printer"
~~~

Show notes for one asset:

~~~bash
deltaaegis asset-notes 'ASSET_KEY'
~~~

Show annotation history for one asset:

~~~bash
deltaaegis asset-notes 'ASSET_KEY' --history
~~~

List saved asset annotations:

~~~bash
deltaaegis asset-annotations
~~~

Limit annotation output:

~~~bash
deltaaegis asset-annotations --limit 10
~~~

Asset annotations are used by investigation reports when an event or alert subject matches a saved annotation.

## Risk Register

DeltaAegis v0.8.5 adds an explainable risk register for prioritizing review.

The risk register uses existing DeltaAegis data:

- delta event severity
- open alerts
- acknowledged alerts
- repeated recent activity
- asset criticality
- missing asset annotations
- asset owner and role context

Show the top risk subjects:

~~~bash
deltaaegis risk
~~~

Limit output:

~~~bash
deltaaegis risk --limit 10
~~~

Show detailed scoring reasons:

~~~bash
deltaaegis risk --details
~~~

Filter by subject text:

~~~bash
deltaaegis risk --subject SUBJECT_KEY
~~~

Show one subject's detailed risk record:

~~~bash
deltaaegis asset-risk SUBJECT_KEY
~~~

Start the read-only local dashboard:

~~~bash
deltaaegis dashboard
~~~

Start the dashboard on a custom local port:

~~~bash
deltaaegis dashboard --host 127.0.0.1 --port 8090
~~~

Start the dashboard with token protection:

~~~bash
deltaaegis dashboard --token CHANGE_ME
~~~

The dashboard is local-first and read-only by default. It should remain bound to `127.0.0.1` unless it is placed behind trusted network controls, a reverse proxy, or other access protection.

Risk scoring is intentionally explainable. Each score is built from visible reasons such as event severity, open alerts, asset criticality, and repeated activity.

## Data Model

DeltaAegis stores its local state in SQLite.

Core data includes:

~~~text
snapshots
asset_observations
service_observations
finding_observations
delta_events
asset_lifecycle
alerts
alert_notes
asset_annotations
asset_annotation_history
~~~

The `delta_events` table preserves the append-only event history.

The `alerts` table maintains stateful operator-facing alerts.

The `alert_notes` table records analyst review decisions.

The `asset_annotations` and `asset_annotation_history` tables preserve asset owner and context information.

## Investigation Dashboard and Asset Inventory

DeltaAegis v0.8.5 expands the read-only dashboard and reporting workflow into a scope-aware investigation console.

### Asset Inventory CLI

List assets observed in a specific network scope:

    deltaaegis assets --scope 192.168.4.0/24

Filter by lifecycle state:

    deltaaegis assets --scope 192.168.4.0/24 --state ACTIVE

Filter by identity type:

    deltaaegis assets --scope 192.168.4.0/24 --identity GLOBAL_MAC

Inspect a specific asset without crossing subnet boundaries:

    deltaaegis asset 192.168.4.203 --scope 192.168.4.0/24
    deltaaegis asset mac:cc:ba:97:64:1e:74 --scope 192.168.4.0/24

### Dashboard Investigation Features

The dashboard now includes:

- Collapsible dashboard cards to reduce visual overload.
- Scope-aware Asset Inventory table.
- Asset Detail panel with lifecycle state, latest observation, services, findings, recent events, alerts, and annotation context.
- Asset selector dropdown for quickly choosing a device.
- Clickable subjects in Asset Inventory, Top Risk Subjects, Delta Events, and Alerts.
- Scope-aware asset APIs.

Useful dashboard APIs:

    /api/assets?scope=192.168.4.0/24&limit=25
    /api/assets?scope=192.168.4.0/24&state=ACTIVE
    /api/assets?scope=192.168.4.0/24&identity=GLOBAL_MAC
    /api/asset?scope=192.168.4.0/24&identifier=192.168.4.203
    /api/asset?scope=192.168.4.0/24&identifier=mac:cc:ba:97:64:1e:74

### Scope-Aware Reports

Generate a report for one subnet scope:

    deltaaegis report --scope 192.168.4.0/24

Control report size:

    deltaaegis report --scope 192.168.4.0/24 --limit 25 --risk-limit 10 --asset-limit 25

Reports now include:

- Network Scope Summary
- Asset Lifecycle Summary
- Asset Inventory
- Top Risk Subjects
- Active Alerts
- Delta Events
- Dashboard and API Usage Notes
- Recommended Next Actions

Scoped reports are isolated so events, alerts, assets, and report context from unrelated network scopes do not mix.

## Scope and Limitations

DeltaAegis v0.8.5 is a network-state monitoring, investigation, risk prioritization, reporting, and dashboard prototype.

It is not a replacement for an enterprise SIEM.

DeltaAegis does not currently:

- Ingest endpoint logs
- Store full packet captures indefinitely
- Perform machine-learning anomaly detection
- Send email or chat notifications
- Provide a write-enabled or remotely exposed dashboard by default
- Automatically discover business owners for assets
- Replace manual analyst review

Its conclusions are limited to the telemetry provided by NetSniper and the local DeltaAegis database.

## Read-Only Dashboard

DeltaAegis v0.8.5 adds a local read-only dashboard for making scan results easier to understand.

Start the dashboard:

~~~bash
deltaaegis dashboard
~~~

Default URL:

~~~text
http://127.0.0.1:8090
~~~

The dashboard includes:

- What am I looking at? explanation panel
- NetSniper scan context
- latest scan and baseline scan comparison
- scan freshness status
- observed asset, IP, and MAC counts
- identity coverage
- top risk subjects
- recent delta events
- recent alerts
- asset annotations
- risk and identity legend
- recommended next steps

Dashboard API endpoints:

~~~text
/api/summary
/api/scan-context
/api/risk
/api/events
/api/alerts
/api/annotations
/healthz
~~~

Security note:

The v0.6.0 dashboard is intentionally read-only. It does not run scans, modify alerts, change annotations, or perform remediation actions. Keep it bound to `127.0.0.1` unless you are using a trusted network, SSH tunnel, reverse proxy, VPN, or token protection.

Token-protected example:

~~~bash
deltaaegis dashboard --token CHANGE_ME
~~~

Then query with:

~~~bash
curl -H 'X-DeltaAegis-Token: CHANGE_ME' http://127.0.0.1:8090/api/summary
~~~
